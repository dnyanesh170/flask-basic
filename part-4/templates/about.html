Understanding Dynamic Routes

Dynamic routes in Flask allow parts of a URL to behave like variables. These values are captured directly from the address bar and passed into Python functions for processing.

Common URL Parameter Types

Pattern Purpose Sample URL
<name> Text value (string) /user/rahul
<int:id> Whole numbers only /post/5
<float:price> Decimal values /price/1499.50
<path:file> Path with slashes /docs/python/basics.pdf
Trailing Slash Tip

Routes defined with a trailing slash (for example /about/) work for both /about and /about/. Flask automatically handles the redirection.

Where Dynamic Routes Are Useful

  • User dashboards like /user/<username>
  • Blog systems such as /post/<int:post_id>
  • Product pages like /product/<int:product_id>
  • Category-based navigation paths
